# Mini App Status Next Action Triggers

## 自動実行ルール
ステータス確認は情報提供が主目的のため、自動実行はしない。
ユーザーの指示に応じて該当Skillを実行する。

## 起動条件テーブル

| ID | 起動条件 | 実行Skill | 優先度 | 備考 |
|----|---------|-----------|--------|------|
| T1 | ユーザーが「続きから」「再開」を指示 | 未完了の次フェーズ | 1 | 中断箇所から再開 |
| T2 | ユーザーが特定フェーズの実行を指示 | 指定されたSkill | 2 | 明示的な指示 |

## ステータス確認後の推奨アクション

### 進捗に応じた提案

| 現在のフェーズ | 次の推奨アクション |
|---------------|-------------------|
| 要件定義完了 | mini-app-design |
| 設計完了 | mini-app-build |
| 実装完了 | mini-app-test |
| テスト完了 | mini-app-review |
| レビュー完了 | mini-app-deploy |
| デプロイ完了 | なし（または改善サイクル） |

## ステータス確認時の報告形式

```markdown
## 開発ステータス: {{app_name}}

**現在のフェーズ**: {{phase}}
**進捗**: {{progress}}%

### 完了済み
- ✅ {{completed_phases}}

### 次のアクション
- {{next_action}}

続きを実行しますか？
```
