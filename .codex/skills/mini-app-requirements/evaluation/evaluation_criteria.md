# Mini App Requirements 評価指標

## 必須チェック項目

### 構造チェック（Pass/Fail）

| 項目 | 基準 | 重要度 |
|------|------|--------|
| テンプレート準拠 | 必須セクション（目的/対象ユーザー/機能要件/非機能要件/技術制約）が存在する | Critical |
| 機能要件定義 | Must機能が最低1つ定義されている | Critical |
| 対象ユーザー明示 | ペルソナまたは利用者が特定されている | Critical |
| 未記載項目の明示 | 不明項目は「未記載」「不明」と明記されている | High |
| GitHub Pages互換 | 技術制約がGitHub Pages要件を満たしている | High |

### 内容チェック（スコアリング）

| 観点 | 評価基準 | 配点 |
|------|----------|------|
| 完全性 | 必須セクションが全て埋まっている | 30 |
| 明確性 | 曖昧な表現（「〜など」「適切に」「必要に応じて」）がない | 25 |
| 実現可能性 | GitHub Pages + 静的サイトで実現可能な要件のみ | 25 |
| 一貫性 | 機能要件と非機能要件に矛盾がない | 20 |

### 許容例外

- 参考情報セクションが空欄（参考がない場合）
- Could機能が未定義（必須ではない）

## 採点基準

- **Pass**: 全Criticalチェック項目がPass かつ スコア80点以上
- **Conditional Pass**: 全Criticalチェック項目がPass かつ スコア60-79点
- **Fail**: CriticalチェックにFailあり または スコア60点未満
